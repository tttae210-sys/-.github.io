# [Cloud] 2주차: AWS 운영 환경 이해 및 CLI 실습

1주차에 클라우드의 개념을 잡았다면 2주차에는 실제 AWS 리소스를 조작하기 위한 다양한 접근 방법과 관리 도구에 대해 학습했습니다.

---

## 2-1. AWS 접근 방법 3가지
AWS 리소스를 관리하고 제어하는 방법은 크게 세 가지로 나뉩니다.

1. **AWS Management Console**: 웹 브라우저 기반의 GUI로, 클릭만으로 서비스 실행과 설정이 가능해 초보자에게 적합합니다.
2. **AWS CLI**: 터미널에서 명령어로 리소스를 제어하며, 스크립트 자동화나 대규모 관리에 유리합니다.
3. **AWS SDK**: Python, Java 등 프로그래밍 언어를 사용하여 애플리케이션 내부에서 리소스를 제어합니다.

---

## 2-2. AWS Management Console 주요 구성
웹 콘솔은 시각적으로 리소스를 관리할 수 있는 도구이며, 주요 메뉴 구성은 다음과 같습니다.

* **서비스 검색창**: "EC2", "S3" 등 원하는 서비스로 바로 이동할 수 있는 필드입니다.
* **리전 선택**: 현재 사용 중인 리전을 확인하고 전환할 수 있습니다. 리전 선택에 따라 생성·조회되는 리소스가 달라지므로 주의해야 합니다.
* **AWS CloudShell**: 브라우저에서 바로 실행 가능한 터미널 환경으로, 별도 설치 없이 CLI 작업을 수행할 수 있습니다.
* **계정 정보**: 로그인한 계정 ID와 사용자 유형(Root/IAM)을 확인할 수 있습니다.

---

## 2-3. 리전 변경과 대시보드 활용
AWS 리소스는 전 세계 여러 리전에 분산 배치됩니다.

* **리전의 중요성**: 서울 리전에서 만든 EC2 인스턴스는 오하이오 리전에서는 조회되지 않습니다. 서비스 대상과 가까운 리전을 선택하는 것이 지연 시간을 줄이는 핵심입니다.
* **서비스 대시보드**: 각 서비스의 현황을 시각적으로 보여주는 요약 페이지입니다. 리소스 상태와 모니터링 지표를 한눈에 파악할 수 있습니다.



---

## 2-4. AWS CLI 및 CloudShell 실습
명령줄 도구인 CLI를 활용하면 GUI보다 빠르게 리소스를 조회하고 자동화할 수 있습니다.

### 핵심 명령어 실습
1. **S3 버킷 목록 조회**: `aws s3 ls`
2. **특정 버킷 내 객체 조회**: `aws s3 ls s3://버킷이름`
3. **EC2 인스턴스 정보 조회**: `aws ec2 describe-instances`
4. **IAM 사용자 목록 조회**: `aws iam list-users`

> **보안 주의사항**: 로컬 PC에 CLI를 설치할 경우 Access Key와 Secret Key가 유출되지 않도록 주의해야 합니다. 공용 컴퓨터라면 로컬 설치 대신 **CloudShell** 사용을 강력히 권장합니다.
